# cic-ids-2018-labeling

Autonomous end-to-end data processing pipeline for the CIC-IDS2018 network intrusion detection dataset.

## Overview

The pipeline reproduces the full ingestion-to-features workflow from raw network captures to labelled, preprocessed feature vectors suitable for machine learning experiments:

| Stage | Description |
|-------|-------------|
| **Ingestion** | Downloads daily PCAP archives from the public AWS S3 bucket `cse-cic-ids2018` |
| **Flow extraction** | Generates per-flow CSV files via CICFlowMeter (requires Java 8) |
| **Labeling** | Assigns per-row attack labels using Apache Spark (map-reduce), driven by `configs/attack_schedule.yaml` |
| **Preprocessing** | Applies feature selection, missing-value imputation, standard scaling, and optional stratified sampling |
| **Cache** | Persists per-day results to `preprocessed_cache/<day>/` to avoid redundant reprocessing |



## Data Sources and Labeling Process

The official CIC-IDS-2018 dataset provides two main types of files for each day/session:

- **Raw PCAP files**: Complete packet captures of network traffic, suitable for custom flow extraction and deep packet analysis.
- **Pre-extracted CSV files for ML**: Flow-level feature matrices generated by the CICFlowMeter tool, but with no attack/benign labels included.

### Flow Extraction with CICFlowMeter

To transform raw PCAPs into feature-rich CSVs, the pipeline uses [CICFlowMeter](https://github.com/UNBCIC/CICFlowMeter). This tool extracts per-flow statistics (e.g., IPs, ports, protocol, timing, packet/byte counts) from the PCAPs. However, **the resulting CSVs do not contain any ground-truth labels** indicating whether a flow is benign or associated with a specific attack.

### Manual Labeling Process

To enable supervised machine learning, the pipeline implements a custom labeling stage. Labels are assigned by cross-referencing each flow's metadata (timestamps, IPs, ports) with the official attack schedule and IP ranges provided in `configs/attack_schedule.yaml`. This process is performed using Apache Spark for scalability and reproducibility.

**Key points:**
- Labeling is not provided by CICFlowMeter or the official CSVs; it must be reconstructed from auxiliary information.
- The accuracy of labeling depends on the precision of the attack schedule and the quality of the extracted flow features.
- This approach enables the creation of ML-ready datasets with explicit attack/benign labels, suitable for research and benchmarking.

For a detailed comparison of the official and improved datasets, and for further research context, see the `CheckDataset.ipynb` notebook.

## Usage

Run with the days configured in `configs/settings.py`:

```bash
python main.py
```

Process specific days:

```bash
python main.py --days Thursday-15-02-2018 Friday-16-02-2018
```

Force reprocessing, ignoring any existing cache:

```bash
python main.py --force
```

Persist the preprocessed DataFrame to disk:

```bash
python main.py --cache
```

### CLI Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--days` | from `settings.py` | Days to process |
| `--force` | `False` | Discard cache and regenerate all outputs |
| `--sample` | `20000` | Maximum sample size per run (0 = no limit) |
| `--cache` / `--no-cache` | `False` | Save `preprocessed.csv` to the cache directory |

## Output Files

Each processed day produces the following files under `preprocessed_cache/<day>/`:

| File | Contents |
|------|----------|
| `benign_records.csv` | Labelled benign network flows |
| `attack_records.csv` | Labelled attack network flows |
| `preprocessed.csv`   | Scaled and encoded feature matrix (only with `--cache`) |
